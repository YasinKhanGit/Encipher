Combinator

Descendant Combinator

The CSS descendant combinator (a single space) is used to select elements inside elements. Elements matched by the second selector are selected if they have an ancestor (parent, parent's parent, parent's parent's parent, etc.) element matching the first selector.

Child Combinator

Selects elements that are direct children of a specified parent.

Written as parent > child between two selectors.

Does not select deeper nested descendants.

Improves styling precision and avoids unintended matches.

Next Sibling Combinator

The + symbol represents the next sibling combinator in CSS. It selects an element that immediately follows another element.

Both elements must share the same parent.

Only the first immediate sibling is matched, not all following siblings. Syntax: A + B means select B if it comes right after A.

It can be combined with class, ID, or attribute selectors.

It is useful for styling elements based on their preceding sibling.

Subsequent-sibling Combinator

  1. The ~ combinator selects elements that share the same parent.
  2. It matches all siblings that come after the first specified element.
  3. The matched elements do not need to be immediately adjacent.
  4. Both elements must be on the same hierarchy level in the DOM.

Syntax: element1 ~ element2 { ... }.

It applies styles to all subsequent siblings of the given type.

It is case-insensitive for HTML element names.

It is useful for styling multiple related elements after a trigger element.
